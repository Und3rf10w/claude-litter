# Deepwork plugin — doc sync rule

**Deepwork doc sync rule**: Editing a skill `SKILL.md` → check `README.md` §10 commands row. Adding a hook → check `README.md` §13 hooks table. Changing `scripts/setup-deepwork.sh` flag parsing → check `README.md` §11 flags table. Adding/updating `references/parallel-execution.md` → check execute-mode skill stance (`profiles/execute/PROFILE.md`) for the citation path.

Never quote skill-prompt, stance, or `PROFILE.md` text in `README.md` or `references/*.md` — docs cite by path, not by restatement. The categorical ban is enforced by the acceptance criteria in `proposals/v2-final.md` §8.5.

**Rule 5 — worktree operational triangle:** Changing the worktree operational recipe in `profiles/execute/stances/executor-stance.md` step 3 → check `references/parallel-execution.md` source-maintenance note (CC version) AND `skills/deepwork/SKILL.md` execute-mode pointer for consistency.
